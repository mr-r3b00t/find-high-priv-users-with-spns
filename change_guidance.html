Active Directory: Disabling RC4 for User Accounts

Report Date: January 30, 2026

Guidance on per-user RC4 disablement, AES key regeneration (including same-password reset), impacts for SPN/service accounts, and rollback/re-enable options. Updated with current Microsoft roadmap and CVE-2026-20833 phase details.

Current Status – Microsoft RC4 Deprecation Roadmap (as of Jan 2026)

Microsoft is phasing out RC4 in Kerberos due to cryptographic weaknesses and vulnerabilities (e.g., CVE-2026-20833 facilitating Kerberoasting). Key milestones:

  • January 2026+: New auditing + temporary registry key RC4DefaultDisablementPhase for opt-in enforcement.
  • April 2026: Enforcement mode begins – RC4 blocked by default on DCs unless explicitly allowed per account/KDC.
  • July 2026: Final enforcement – registry key removed; RC4 disabled domain-wide by default on Server 2008+ DCs (exceptions require explicit per-account config).
Recommendation: Proactively disable RC4 on service accounts now, regenerate AES keys, and prepare for mid-2026 enforcement to avoid outages.

Disabling RC4 on a Specific User Account

Modify msDS-SupportedEncryptionTypes bitmask (4=RC4, 8=AES128, 16=AES256). Set to 24 (AES only) to disable RC4.

PowerShell Method

# Disable RC4 (enable AES only)
Set-ADUser -Identity "svc-account" -Replace @{'msDS-SupportedEncryptionTypes'=24}

# Reset password to generate AES keys (see same-password technique below)

GUI Method

  1. ADUC: User Properties → Account tab → Check AES 128/256 boxes.
  2. ADSI Edit: Set msDS-SupportedEncryptionTypes to decimal 24 (hex 0x18).
  3. Reset password.

Resetting Password to the SAME Value – Minimal Disruption

Resetting to the identical password regenerates Kerberos keys (including AES) without changing credentials. Ideal for service accounts.

Benefit: NTLM hash unchanged; Kerberos keys updated in supplementalCredentials. Services continue after ticket renewal.

PowerShell Example

$pwd = "ExistingStrongP@ssw0rd123!"  # From vault/owner
$securePwd = ConvertTo-SecureString $pwd -AsPlainText -Force

Set-ADAccountPassword -Identity "svc-account" -Reset -NewPassword $securePwd

# Verify
Get-ADUser -Identity "svc-account" -Properties PasswordLastSet, msDS-SupportedEncryptionTypes
  • Replicate AD: repadmin /syncall
  • Monitor events (ID 3/27) on DCs.
  • Rare: Reset twice for old accounts.
Caution: Test first. Legacy AES-incompatible clients will still fail.

Rollback and Re-Enable Options

If disabling RC4 causes issues (e.g., legacy service failures), rollback is straightforward per account. Domain-wide rollback depends on the current phase.

Per-Account Rollback (Re-Enable RC4)

Revert by including RC4 in the bitmask (add 4). Common values:

Value (Decimal)HexEncryption Types EnabledUse Case
280x1CAES128 + AES256 + RC4Re-enable RC4 while keeping AES (recommended interim)
40x04RC4 onlyLegacy-only fallback (temporary!)
310x1FAll (AES128/256 + RC4 + others)Max compatibility (not recommended long-term)
# Re-enable RC4 (AES + RC4)
Set-ADUser -Identity "svc-account" -Replace @{'msDS-SupportedEncryptionTypes'=28}

# Optional: Reset password again to ensure keys regenerate
# ... (same as above)

After change, reset password if needed. This works even after domain-wide enforcement (July 2026+), as Microsoft allows explicit per-account RC4 exceptions indefinitely (though strongly discouraged long-term).

Domain-Wide Rollback (Pre-July 2026)

While RC4DefaultDisablementPhase registry exists (on DCs: HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\Kerberos\Parameters):

  • Set to 0 or delete → Reverts to legacy (RC4 allowed by default; audit off).
  • Set to 1 → Audit mode only (logs RC4 but allows it).
  • Restart KDC service or reboot DC for change to apply.
Post-July 2026: Registry key removed in updates; no domain-wide rollback via registry. Use per-account msDS-SupportedEncryptionTypes exceptions only. Plan legacy remediation to avoid permanent exceptions.
Best Practice: Treat re-enables as temporary. Audit RC4 usage post-rollback and plan upgrades for incompatible systems.

Impacts for Accounts with SPNs (Service Accounts)

Summary & Recommendations

  • Set msDS-SupportedEncryptionTypes = 24 per service account.
  • Use same-password reset for AES key generation without disruption.
  • Rollback: Revert bitmask to include 4 (e.g., 28); pre-July 2026 use registry for domain rollback.
  • Modern envs: Seamless; legacy: Upgrade or temporary exceptions.
  • Prepare for July 2026 – RC4 off by default; explicit config required for exceptions.
